# 1000201 任务怪物未生成修复

## 问题

主线 `1000201 - 漩涡止于中心` 中，子任务 `100020104` 完成后进入 `100020105`，客户端任务目标已经切换，但地图和场景中没有生成需要击杀的怪物。

相关配置来自：

- `resources/Config/Level/Mission/1000201/MissionInfo_1000201.json`
- `resources/Config/Level/Mission/1000201/MissionInfo_1000201.partial.json`

`100020105` 的怪物配置：

```json
{
  "ID": 100020105,
  "FinishType": "KillMonster",
  "ParamInt1": 16,
  "ParamInt2": 200001,
  "WayPointType": "Monster",
  "WayPointGroupID": 16,
  "WayPointEntityID": 200001
}
```

## 根因

服务端会先读取 `MissionInfo_1000201.json`，如果存在 `MissionInfo_1000201.partial.json`，会使用 partial 文件中的 `SubMissionList` 覆盖完整配置中的子任务列表。因此 `100020105` 的配置实际会被加载。

问题不在配置读取，而在任务切换刷怪流程：

1. `100020104` 完成后，服务端接取 `100020105`。
2. `100020105` 需要根据任务配置加载 group `16` 中的怪物 `200001`。
3. 旧逻辑只处理 `GroupIDList`，没有从 `KillMonster.ParamInt1` 或 `WayPointGroupID` 自动推导任务怪物 group。
4. 部分情况下 group `16` 已经提前存在于服务端场景实体表中，任务开始时再次发送已有实体，客户端不会稳定显示怪物。

## 修复

本次修复做了三件事：

1. 在 `SubMissionInfo` 中补充 waypoint 字段解析：
   - `WayPointType`
   - `WayPointFloorID`
   - `WayPointGroupID`
   - `WayPointEntityID`

2. 新增 `GetAutoLoadGroupIds()`，自动从以下位置推导需要加载的任务 group：
   - `GroupIDList`
   - `FinishType == KillMonster` 时的 `ParamInt1`
   - `WayPointType == "Monster"` 时的 `WayPointGroupID`

3. 子任务切换时，先同步新任务状态，再刷新任务怪物 group：
   - `100020104 MISSION_FINISH`
   - `100020105 MISSION_DOING`
   - 刷新 group `16`

如果目标 group 已经存在，使用 `RefreshGroup()` 先移除旧实体，再重新创建并发送刷新包，确保怪物是在任务开始后真正生成。

- `Common/Data/Config/MissionInfo.cs`
- `GameServer/Game/Mission/MissionManager.cs`
- `GameServer/Game/Scene/SceneEntityLoader.cs`
- `Tests/MissionConfigBehaviorTests/`

实机验证结果：

## 后续修正

第一次修复时，任务开始后使用 `RefreshGroup()` 刷新了整个 group，导致 group `16` 内的 NPC 和机关也被一起显示出来。

后续改为任务触发时只刷新目标怪物：

- `FinishType == KillMonster` 时使用 `ParamInt2` 作为目标怪物 `instId`。
- `WayPointType == "Monster"` 时使用 `WayPointEntityID` 作为目标怪物 `instId`。
- 对 `100020105` 来说，只刷新 `groupId=16, instId=200001` 这一只怪物。
- 刷新前会移除 group 中已经提前生成的旧实体，避免残留多余 NPC/机关。
